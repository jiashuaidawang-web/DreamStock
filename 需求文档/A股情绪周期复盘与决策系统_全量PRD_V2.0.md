# 0. 文档说明

## 0.1 本版定位

本文档是《A股情绪周期复盘与决策系统》的全量 PRD，不再按“一期、二期、三期”拆分。本文档按完整产品形态一次性定义：产品目标、用户角色、业务边界、信息架构、功能模块、数据指标、评分模型、页面需求、用户故事、验收标准、数据模型、接口需求、权限合规与非功能要求。

本系统不是传统行情软件，也不是单纯的看盘工具，而是把《顿悟股道》中的“周期、情绪、资金、题材、龙头、趋势、心态、复盘”抽象为一套可记录、可计算、可回测、可复盘、可纠偏的金融交易研究工作台。

## 0.2 产品原则

1. **以周期为根**：任何个股分析、板块分析、交易计划都必须先落到市场所处周期与情绪阶段。
2. **以大局观为先**：先判断大盘趋势结构、资金供需、政策消息、市场情绪，再判断题材、板块、龙头和个股。
3. **以主线为抓手**：题材不是孤立标签，而是资金合力、市场共识、赚钱效应和龙头带动的结果。
4. **以龙头为核心观察对象**：龙头不是静态名单，而是在周期、情绪、空间高度、板块强度、换手分歧中动态竞争出来的市场地位。
5. **以复盘为闭环**：用户每天的判断、计划、操作、情绪、纪律执行、盈亏结果和后验验证必须形成完整闭环。
6. **以风控合规为底线**：系统提供研究、复盘、记录、规则校验与风险提示，不承诺收益，不替代用户独立决策，不输出确定性买卖指令。

## 0.3 需求范围

全量范围包含：行情数据接入、市场周期识别、每日情绪打分、资金供需监控、政策消息标注、题材主线识别、涨停板生态分析、龙头竞争与梯队分析、趋势龙头识别、个股与市场情绪相关性分析、交易计划、个人操作复盘日志、心态纪律评估、案例库、规则配置、回测验证、数据质量、权限与合规。

## 0.4 非目标

- 不做证券投顾业务，不提供“买入/卖出/持有”的确定性投资建议。
- 不做收益承诺、排名诱导、荐股群导流、跟单交易。
- 不把单一指标包装成万能信号。
- 不把主观笔记直接等同于交易指令。
- 不允许在没有数据来源、没有评分依据、没有计算链路的情况下生成假信号。

# 1. 产品目标

## 1.1 产品愿景

建设一套面向 A 股短线、超短线、题材周期、龙头战法、趋势战法交易者的“情绪周期复盘与决策研究系统”。系统把每日市场从“看感觉、看截图、看别人复盘”升级为“看周期、看情绪、看资金、看主线、看龙头、看自我行为偏差”。

最终目标是让用户每天完成四件事：

1. **看清市场**：当前是冰点、修复、启动、发酵、高潮、分歧、退潮，还是熊末牛初、牛市主升、牛尾风险。
2. **看清主线**：市场资金真正围绕哪些题材、板块、风格、指数方向形成合力。
3. **看清个股地位**：某只股票到底是总龙头、板块龙、趋势龙、补涨、跟风、套利、掉队，还是风险标的。
4. **看清自己**：自己今天是否尊重周期、是否追高、是否贪胜、是否无计划交易、是否违背止损、是否陷入个股执念。

## 1.2 业务目标

| 目标编号 | 目标 | 业务解释 | 产品化抓手 |
|---|---|---|---|
| G1 | 建立每日情绪标准化评分 | 把涨停、跌停、连板、高标、量能、赚钱效应、亏钱效应、指数环境量化 | 情绪总分、分项评分、等级、阶段 |
| G2 | 建立周期波动可视化 | 让用户看到情绪与趋势不是一天的点，而是连续波动的线 | 周期趋势图、阶段色带、拐点标记 |
| G3 | 建立题材主线识别 | 判断市场是在主线抱团、轮动混战、退潮试错还是无主线 | 题材热度、主线强度、板块梯队 |
| G4 | 建立龙头动态竞争模型 | 跟踪龙头从首板试错、二板筛选、三板确认、四五板空间博弈到分歧退潮 | 龙头排名、梯队结构、分歧一致状态 |
| G5 | 建立趋势龙头识别模型 | 识别机构趋势牛股、长期均线、平台突破、RS 强度、周线确认日线操作 | 趋势评分、突破监控、波段计划 |
| G6 | 建立个股与大盘情绪相关性 | 判断个股是否顺周期共振、逆势穿越、弱转强、补涨、掉队 | 相关系数、Beta、情绪敏感度、背离标签 |
| G7 | 建立个人交易复盘闭环 | 记录计划、操作、情绪、纪律、盈亏、复盘结论 | 交易日志、行为标签、偏差统计 |
| G8 | 建立策略验证与案例沉淀 | 把历史行情、龙头案例、失败案例转为可检索的学习资产 | 案例库、回放、模板、复盘报告 |

## 1.3 用户价值

- **盘后 10 分钟看清市场全局**：不用翻几十个榜单，直接看到周期、情绪、资金、主线、龙头状态。
- **盘前形成交易计划**：根据昨日复盘和今日风险日历，形成可执行计划，不临盘乱追。
- **盘中有纪律提醒**：当用户操作偏离计划、追入非主线、情绪高潮追高、退潮期加仓时，系统给出风险提示。
- **盘后复盘可沉淀**：每笔交易都有前因后果和后验验证，避免重复犯错。
- **从个股思维升级为周期思维**：系统强制让用户先回答“今天是什么周期、什么情绪、什么主线”，再看个股。

# 2. 用户角色

## 2.1 角色总览

| 角色 | 核心诉求 | 高频场景 | 成功标准 |
|---|---|---|---|
| 短线交易者 | 快速判断今天能不能做、做什么方向、仓位多大 | 盘后复盘、盘前计划、盘中盯主线 | 减少退潮期重仓、高潮期盲追 |
| 龙头战法交易者 | 找到总龙、板块龙、补涨龙，判断分歧和一致 | 连板梯队、涨停板复盘、题材发酵 | 能识别龙头地位变化和风险切换 |
| 趋势交易者 | 寻找机构趋势龙头，做波段和持仓管理 | 周线筛选、平台突破、RS 强度跟踪 | 能过滤弱趋势和假突破 |
| 复盘研究者 | 建立历史周期、题材、龙头、案例库 | 周期回放、案例标注、复盘报告 | 形成可复用交易认知 |
| 交易教练/团队负责人 | 统一复盘标准，检查成员纪律 | 团队复盘、交易日志抽查、行为偏差统计 | 发现高风险交易习惯 |
| 数据管理员 | 确保行情、题材、榜单、财务、新闻数据质量 | 数据对账、异常修正、规则配置 | 数据准确、可追溯 |
| 合规管理员 | 控制内容边界与风险提示 | 策略发布、报告分享、用户权限 | 不越过投顾合规红线 |

## 2.2 用户画像

### 2.2.1 新手短线用户

特点：容易看单只股票，不会看周期；容易被涨停、新闻、群消息影响；交易日志不完整。

系统价值：强制从大盘周期、情绪阶段、主线强弱、仓位建议区间、风险标签开始，降低盲目交易。

### 2.2.2 成熟短线用户

特点：懂涨停、懂题材、懂连板，但复盘依赖主观经验，缺少结构化沉淀。

系统价值：提供可量化指标、历史相似行情、龙头竞争图谱、操作偏差统计，提升复盘效率。

### 2.2.3 趋势交易用户

特点：关注中线趋势、行业景气、机构抱团、平台突破和相对强度。

系统价值：提供趋势龙头池、长期均线、RS 强度、周线确认、平台突破、波段计划。

### 2.2.4 团队研究用户

特点：多人协作复盘，需要统一术语、统一模板、统一结论沉淀。

系统价值：提供团队空间、模板、标注、评论、复盘报告、权限与审计。

## 3.2 核心业务链路

### 3.2.1 每日复盘链路

数据采集 -> 数据清洗 -> 情绪指标计算 -> 周期阶段识别 -> 题材主线识别 -> 龙头梯队分析 -> 风险标签生成 -> 用户复盘填写 -> 日报生成 -> 次日计划。

### 3.2.2 个股研究链路

输入股票代码 -> 拉取行情与板块归属 -> 计算相对强度 -> 识别题材地位 -> 计算与大盘情绪相关性 -> 判断龙头/补涨/跟风/掉队 -> 展示历史相似案例 -> 写入关注池或交易计划。

### 3.2.3 交易行为闭环

盘前计划 -> 盘中操作 -> 自动匹配计划 -> 盘后复盘 -> 后验验证 -> 行为偏差归因 -> 生成改进动作 -> 次日提醒。

| 连板数量 | consecutive_limit_count | int | 二连板及以上数量 | 越高越强 |
| 最高连板高度 | max_board_height | int | 市场空间高度 | 越高越强，但极端高位需风险标签 |
| 首板数量 | first_board_count | int | 新题材试错活跃度 | 修复期重要 |
| 炸板率 | failed_limit_rate | decimal | 炸板数 / 触板数 | 越高越弱 |
| 大面数量 | big_loss_count | int | 昨日涨停/高标今日大跌数 | 越高越弱 |
| 晋级率 | promotion_rate | decimal | 昨日连板今日继续连板比例 | 越高越强 |
| 涨跌家数比 | adv_dec_ratio | decimal | 上涨家数 / 下跌家数 | 越高越强 |
| 成交额 | total_turnover | decimal | 全市场成交金额 | 与市值和位置共同判断 |
| 量能变化率 | turnover_change_rate | decimal | 较昨日或均值变化 | 放量修复为强，缩量冲高为弱 |
| 主线集中度 | theme_concentration | decimal | 前三题材涨停占比 | 适中到高代表主线明确 |
| 高标反馈 | high_board_feedback | enum | 高标次日溢价/亏钱状态 | 判断接力环境 |

## 5.3 周期阶段枚举

| 阶段 | code | 典型特征 | 操作含义 |
|---|---|---|---|
| 极寒冰点 | EXTREME_ICE | 跌停多、炸板高、连板断裂、成交低迷 | 以观察为主，等待修复信号 |
| 冰点修复 | ICE_REPAIR | 跌停减少、首板回暖、指数止跌 | 小仓位试错，关注新题材首板 |
| 启动试错 | START_TRIAL | 二板增多、题材出现、赚钱效应初现 | 关注一进二、二进三 |
| 情绪发酵 | EMOTION_FERMENT | 主线明确、连板梯队完整、龙头上位 | 顺主线参与核心标的 |
| 情绪高潮 | EMOTION_CLIMAX | 高标加速、缩量一致、散户亢奋 | 防追高，关注分歧风险 |
| 高位分歧 | HIGH_DIVERGENCE | 龙头开板换手、后排分化、炸板增多 | 只看核心，降低后排参与 |
| 退潮杀跌 | EBB_KILL | 高标补跌、大面扩散、跌停增加 | 防守，减少交易 |

| 熊末牛初 | BEAR_END_BULL_START | 恐慌后放量修复、政策资金共振 | 重点观察趋势龙和情绪龙 |

# 6. 核心评分模型

## 6.1 情绪总分

情绪总分采用 0-100 分，默认由 8 个分项组成。系统必须支持权重配置、版本管理、回测校验和人工修正。

| 分项 | 默认权重 | 说明 |
|---|---:|---|
| 涨停与连板强度 | 20% | 涨停数、连板数、最高板、晋级率 |
| 赚钱效应 | 15% | 涨幅中位数、强势股溢价、昨日涨停表现 |
| 亏钱效应 | 15% | 跌停、大面、炸板、断板亏损 |
| 题材主线强度 | 15% | 主线集中度、梯队完整度、题材持续性 |
| 指数趋势环境 | 10% | 指数涨跌、均线、关键位置、波动 |
| 资金供需强度 | 10% | 成交额、量能变化、资金流入流出 |
| 高标反馈 | 10% | 龙头溢价、高标断板反馈、空间高度 |
| 政策消息催化 | 5% | 重大政策、事件、公告对风险偏好的影响 |

### 6.1.1 评分等级

| 分数区间 | 等级 | 解释 | 风险提示 |
|---|---|---|---|
| 0-20 | 极弱 | 亏钱效应极强，情绪冰点或杀跌 | 禁止重仓试错 |
| 21-40 | 偏弱 | 修复不稳定，题材持续性差 | 仅观察核心信号 |
| 41-60 | 中性 | 混沌轮动或弱修复 | 控制仓位，重视确认 |
| 61-80 | 偏强 | 主线明确，赚钱效应较好 | 关注分歧中的核心机会 |
| 81-100 | 极强 | 情绪高潮或牛市共振 | 同时提示高潮追高风险 |

## 6.2 周期阶段识别规则

周期阶段不能只由当天分数决定，必须结合近 N 日趋势、拐点和结构条件。

### 6.2.1 输入

- 最近 5/10/20/60 日情绪分。
- 最近 5/10/20 日成交额均值与变化率。
- 涨停、跌停、连板、炸板、大面、晋级率。
- 指数趋势位置：5/10/20/60 日均线、关键高低点、平台突破。
- 主线集中度和题材持续天数。
- 高标反馈与空间高度。

### 6.2.2 输出

- 当前周期阶段。
- 阶段置信度。
- 阶段变化方向：上升、下降、震荡、反转待确认。
- 关键证据列表。
- 与昨日相比变化原因。
- 系统阶段与人工修正阶段差异。

### 6.2.3 阶段确认原则

1. 连续性原则：阶段变化至少需要 2 个交易日或强触发条件确认。
2. 结构优先原则：指数破位、量能断裂、高标崩塌时，情绪分再高也必须打风险标签。
3. 主线优先原则：没有持续主线的高分，优先识别为轮动或混沌，而不是强发酵。
4. 资金校验原则：没有成交额配合的修复，不得直接判定为强启动。
5. 高标反馈原则：最高板和核心龙头反馈恶化时，必须提示接力风险。

## 6.3 龙头评分

| 分项 | 权重 | 指标 |
|---|---:|---|
| 空间高度 | 20% | 连板高度、阶段涨幅、突破空间 |
| 板块带动 | 20% | 所属题材涨停数、板块涨幅、梯队完整度 |
| 市场辨识度 | 15% | 成交额、关注度、新闻热度、龙虎榜 |
| 换手健康度 | 15% | 换手率、成交量、是否过度一字 |
| 分歧承接 | 15% | 开板后回封、分歧日承接、次日反馈 |
| 周期共振 | 10% | 所处情绪阶段是否支持继续拓展 |
| 风险扣分 | -20% | 高位缩量一致、监管风险、退潮期、后排亏钱扩散 |

## 6.4 趋势龙头评分

| 分项 | 权重 | 指标 |
|---|---:|---|
| 长期趋势 | 20% | 站上 120/250 日均线，均线多头排列 |
| 平台突破 | 15% | 历史平台突破、放量突破、回踩不破 |
| 相对强度 RS | 20% | 相对指数、行业、全市场排名 |
| 量价结构 | 15% | 放量上涨、缩量回调、筹码稳定 |
| 基本面/产业逻辑 | 10% | 行业景气、业绩改善、政策支持 |
| 周线确认 | 10% | 周线趋势向上，日线用于操作 |
| 市值适配 | 5% | 市值、流动性、机构容量 |
| 风险扣分 | -15% | 高位放量大阴、跌破关键均线、行业退潮 |

## 6.5 个股相关性分析模型

系统需要计算个股与三类对象的关系：

1. 与大盘指数：判断系统性风险和市场 Beta。
2. 与情绪分：判断该股是否情绪敏感、是否逆势穿越。
3. 与所属题材：判断是否板块核心、跟风或掉队。

### 6.5.1 输出标签

| 输出 | 说明 |
|---|---|
| 市场共振 | 个股上涨与指数、情绪、板块同步增强 |
| 情绪敏感 | 个股涨跌高度依赖情绪分变化 |
| 独立走强 | 市场弱但个股强，需要判断是否新龙或庄股风险 |
| 板块核心 | 板块上涨时个股涨幅、成交、辨识度领先 |
| 后排跟风 | 板块强但个股滞后或无带动性 |
| 风险背离 | 个股高位上涨但情绪分、板块、资金走弱 |
| 掉队确认 | 板块继续强而个股走弱，或晋级失败 |

### 7.1.5 用户故事

- 作为短线交易者，我希望打开系统就看到今日情绪分和周期阶段，以便判断明天是否适合进攻。
- 作为龙头战法用户，我希望在首页看到市场总龙和板块龙，以便快速定位核心观察对象。
- 作为复盘用户，我希望首页能提示我今天必须复盘的风险事件，比如高标断板、大面扩散、主线切换。

### 7.1.6 验收标准

- 当日 15:30 后，系统应能生成完整市场快照。
- 首页所有核心卡片均可下钻到对应明细页面。
- 情绪总分、周期阶段、主线题材、龙头候选必须有计算时间和数据版本。

## 7.2 P02 每日情绪打分

### 7.2.1 功能目标

把市场情绪从主观描述变为可解释分数，同时保留人工复盘修正能力。

### 7.2.2 页面功能

1. 情绪总分仪表盘。
2. 八大分项评分雷达图。
3. 涨停/跌停/连板/炸板/晋级明细表。
4. 与昨日、近 5 日、近 20 日对比。
5. 系统评分解释。
6. 人工修正入口。
7. 评分版本与历史记录。

### 7.2.3 评分解释模板

系统生成解释时必须包含：

- 今日情绪处于哪个区间。
- 哪些分项拉高了分数。
- 哪些分项拖累了分数。
- 是否存在背离。
- 对明日复盘关注点。

解释示例结构：

> 今日情绪总分为 X，处于“偏强/偏弱”等级。主要支撑来自涨停数量增加、主线集中度提升；主要压力来自炸板率偏高、高标反馈转弱。由于成交额未同步放大，系统将该日标记为“缩量发酵，需防分歧”。

### 7.2.4 业务规则

- 人工修正不得覆盖原始分数，只能新增人工版本。
- 人工修正必须填写原因。
- 同一交易日可存在多个评分模型版本，但默认展示当前启用版本。
- 如果关键数据缺失超过阈值，情绪分状态为“待确认”。

### 7.3.1 功能目标

展示近 5/20/60/120/250 个交易日情绪分、周期阶段、指数、成交额、主线变化的连续走势。

## 7.4 P04 大盘趋势结构

### 7.4.1 功能目标

把“先看大势”落实为可执行页面，避免用户直接陷入个股思维。

### 7.4.2 功能点

1. 指数趋势：上证、深成指、创业板、科创、北证、全 A。
2. 均线状态：5/10/20/60/120/250 日。
3. 趋势结构：上涨、下跌、震荡、平台突破、平台破位。
4. 关键位置：近期高点、近期低点、缺口、平台、箱体。
5. 技术周期：牛初、牛中、牛末、熊初、熊中、熊末、震荡。
6. 指数与情绪关系：指数强情绪弱、指数弱情绪强、指数情绪共振。

### 7.4.3 关键规则

- 指数破位且情绪退潮时，系统应提升防守权重。
- 指数横盘但题材强时，系统标记为结构性机会。
- 指数强但题材无主线时，系统标记为指数行情或权重行情。
- 指数弱但龙头强时，系统标记为逆势穿越观察，不得直接判定为全面机会。

## 7.5 P05 资金供需监控

### 7.5.1 功能目标

验证市场是否“有钱来”，避免技术形态和情绪信号脱离资金支撑。

### 7.5.2 指标

| 指标 | 说明 |
|---|---|
| 全市场成交额 | 核心量能指标 |
| 成交额分位数 | 当前成交额在近 1 年/3 年的位置 |
| 成交额变化 | 较 5/20 日均值变化 |
| ETF 净流入 | 宽基 ETF、行业 ETF |
| 融资余额变化 | 杠杆资金风险偏好 |
| 北向/外资指标 | 如数据源可用则展示 |
| 新股扩容 | IPO、再融资、解禁、减持压力 |
| 板块成交占比 | 资金是否集中于主线 |

### 7.5.3 业务规则

- 资金供需强弱必须结合市场总市值和位置，不得只看绝对成交额。
- 如果情绪分上升但成交额下降，必须标记“缩量修复/缩量高潮”。
- 如果指数上涨主要由权重驱动但短线亏钱效应扩散，必须标记“指数与短线背离”。

## 7.6 P06 政策消息与事件

### 7.6.1 功能目标

记录外部事件如何影响市场心理、预期、题材和资金风险偏好。

### 7.6.2 功能点

- 政策事件列表。
- 公司公告和行业新闻。
- 事件影响范围：指数、板块、题材、个股。
- 事件类型：政策、监管、产业、业绩、突发、国际、流动性。
- 情绪影响标注：正向、负向、中性、待确认。
- 事件后 1/3/5/10 日效果追踪。

### 7.6.3 业务规则

- 消息只能作为催化，不得单独生成交易结论。
- 事件影响必须在后验中验证，未验证前标记为“预期”。
- 同一事件可关联多个题材和个股。

## 7.7 P07 题材主线矩阵

### 7.7.1 功能目标

识别市场真正的主线、次主线、轮动题材和一日游题材。

### 7.7.2 页面字段

| 字段 | 说明 |
|---|---|
| 题材名称 | 概念/行业/事件主题 |
| 题材热度分 | 综合涨停、涨幅、成交、新闻、持续性 |
| 涨停数量 | 当日题材内涨停数 |
| 连板数量 | 题材内二板及以上数量 |
| 龙头股 | 题材当前核心 |
| 梯队完整度 | 首板、二板、三板、高标结构 |
| 持续天数 | 连续活跃天数 |
| 资金集中度 | 题材成交额占比 |
| 情绪阶段 | 试错、发酵、高潮、分歧、退潮 |
| 风险标签 | 一日游、后排亏钱、龙头断板 |

### 7.7.3 主线判定规则

一个题材被判定为主线，需要同时满足：

- 涨停数量或涨幅贡献居前。
- 存在清晰龙头或核心趋势股。
- 至少具备 2 个交易日持续性，或首日极强并有重大政策/事件催化。
- 梯队结构相对完整。
- 资金集中度明显高于其他题材。

### 7.7.4 题材状态

| 状态 | 说明 |
|---|---|
| 试错 | 首板为主，尚无稳定梯队 |
| 发酵 | 二板、三板增加，板块扩散 |
| 主升 | 龙头带队，梯队完整，赚钱效应强 |
| 高潮 | 后排补涨、一致加速、情绪亢奋 |
| 分歧 | 龙头开板、后排掉队、炸板增加 |
| 退潮 | 龙头补跌，板块亏钱效应扩散 |
| 轮动 | 多题材快速切换，无持续核心 |

## 7.8 P08 涨停板生态

### 7.8.1 功能目标

把涨停板从“列表”升级为“市场情绪生态”。

### 7.8.2 功能点

- 首板池。
- 一进二观察池。
- 二进三确认池。
- 三进四高标池。
- 四五板空间压力池。
- 最高板与市场空间。
- 炸板与回封分析。
- 跌停与大面分析。
- 昨日涨停今日表现。
- 连板晋级率。

### 7.8.3 业务规则

- 首板用于题材试错，不能直接等同龙头。
- 二板用于筛选题材强度，需结合板块联动。
- 三板及以上用于确认市场地位，但必须结合周期阶段和分歧承接。
- 四五板以后必须增加空间压力和监管风险提示。
- 高标断板后的反馈是判断退潮的重要依据。

## 7.9 P09 龙头竞争与梯队

### 7.9.1 功能目标

识别龙头从“候选”到“确认”再到“退位”的全过程。

### 7.9.2 龙头生命周期

| 生命周期 | 说明 | 系统动作 |
|---|---|---|
| 候选 | 首板/二板出现题材辨识度 | 加入候选池 |
| 竞争 | 多只个股争夺板块地位 | 展示竞争关系 |
| 确认 | 高度、带动性、辨识度领先 | 标记板块龙/市场龙 |
| 加速 | 缩量、一字、快速拉升 | 标记一致风险 |
| 分歧 | 开板、放量、换手 | 观察承接 |
| 修复 | 分歧后继续走强 | 提升龙头评分 |
| 退位 | 断板、补跌、带动性消失 | 移入历史龙头 |

### 7.9.3 龙头竞争图谱

- 节点：个股。
- 边：同题材竞争、同高度竞争、同概念关联、资金替代关系。
- 节点大小：成交额或关注度。
- 节点颜色：状态。
- 边粗细：关联强度。

### 7.9.4 业务规则

- 总龙必须具备跨板块或全市场情绪影响，不得只凭高度判定。
- 板块龙必须能带动板块，不带动板块的高标只能标记为独立妖股或资金抱团。
- 补涨龙必须关联原龙头空间打开后的低位扩散。
- 龙头退位后，系统必须记录退位原因：断板、大面、监管、题材退潮、资金切换。

## 7.10 P10 分歧一致分析

### 7.10.1 功能目标

识别“分歧”和“一致”的状态，辅助用户理解换手、加速、缩量、一字板、炸板和回封。

### 7.10.2 指标

| 指标 | 说明 |
|---|---|
| 开板次数 | 一致被打破次数 |
| 回封强度 | 开板后回封速度和封单质量 |
| 成交额放大 | 分歧时是否有承接 |
| 换手率 | 筹码交换程度 |
| 封单金额 | 一致强度参考 |
| 缩量程度 | 加速或风险 |
| 次日溢价 | 分歧/一致后的反馈 |

### 7.10.3 状态判定

| 状态 | 说明 |
|---|---|
| 健康分歧 | 放量换手后仍能回封，板块未崩 |
| 弱分歧 | 开板后承接弱，回封困难 |
| 高位一致 | 缩量加速，情绪亢奋 |
| 一致转分歧 | 高位缩量后放量开板 |
| 分歧转一致 | 换手后再度加速 |
| 退潮分歧 | 分歧后无法修复，亏钱效应扩散 |

## 7.11 P11 个股表现与大盘情绪相关性分析

### 7.11.1 功能目标

帮助用户判断个股不是孤立上涨，而是在大盘、情绪、题材、资金中的相对位置。

### 7.11.2 输入

- 股票代码。
- 分析周期：5/10/20/60/120 日。
- 对比对象：指数、情绪分、所属题材、龙头池。

### 7.11.3 输出

| 输出项 | 说明 |
|---|---|
| 收益相关系数 | 个股收益与指数/情绪/板块收益相关 |
| 情绪敏感度 | 情绪分变化对个股涨跌影响 |
| 相对强度排名 | 个股在全市场和板块中的强度排名 |
| 板块地位 | 龙头、补涨、跟风、掉队 |
| 背离诊断 | 强背离、弱背离、风险背离 |
| 历史相似案例 | 类似走势和情绪环境下的案例 |
| 结论标签 | 共振、独立、补涨、跟风、掉队、风险 |

### 7.11.4 业务规则

- 个股相关性分析必须展示计算区间和样本数量。
- 对新股、停牌、极端一字板等样本不足情况必须提示。
- 相关性高不代表因果关系，页面必须显示解释边界。
- 系统不得输出“必涨、必跌”等确定性表述。

## 7.12 P12 趋势龙头池

### 7.12.1 功能目标

识别机构趋势龙头，区别于游资短线妖股。

### 7.12.2 筛选条件

| 条件 | 说明 |
|---|---|
| 长期均线 | 站上 120/250 日均线或长期均线走平向上 |
| 平台突破 | 突破历史平台或长期箱体 |
| RS 强度 | 相对市场和行业显著走强 |
| 量价结构 | 放量突破、缩量回调、趋势不破 |
| 周线确认 | 周线趋势向上，日线用于操作 |
| 基本面/产业 | 行业景气、政策支持、业绩逻辑或机构关注 |

### 7.12.3 功能点

- 趋势龙头列表。
- 多周期 K 线。
- RS 排名曲线。
- 平台突破提醒。
- 周线确认状态。
- 波段计划。
- 趋势破坏预警。

## 7.13 P13 周期选股策略

### 7.13.1 功能目标

根据不同牛熊阶段给出不同筛选逻辑，避免同一套规则套用所有市场。

### 7.13.2 阶段策略

| 阶段 | 策略重点 | 过滤规则 |
|---|---|---|
| 熊市主跌 | 风险回避，观察超跌和政策信号 | 默认不生成进攻清单 |
| 熊市末期 | 恐慌后关注率先走强、政策资金共振 | 过滤弱反弹和无量反弹 |
| 熊末牛初 | 趋势龙、情绪龙、主线初现 | 提升突破、RS、首批龙头权重 |
| 牛市初期 | 抓主线、抓趋势领涨、抓总龙 | 避免后排低质跟风 |
| 牛市中期 | 主线轮动、趋势波段、龙头切换 | 关注高位分歧和风格切换 |
| 牛市末期 | 防高位泡沫和补跌 | 降低仓位建议，强化止盈规则 |
| 混沌轮动 | 轻仓套利，快进快出 | 禁止格局后排 |

## 7.14 P14 交易计划中心

### 7.14.1 功能目标

让用户在交易前先写计划，交易中按计划执行，交易后按计划复盘。

### 7.14.2 计划字段

| 字段 | 说明 |
|---|---|
| 计划日期 | 交易计划适用日期 |
| 市场阶段判断 | 用户对周期阶段的判断 |
| 系统阶段 | 系统识别阶段 |
| 关注题材 | 主线/次主线/观察题材 |
| 关注个股 | 候选股票 |
| 交易理由 | 大势、题材、龙头、技术、资金、消息 |
| 入场条件 | 价格、分歧、回封、突破、回踩等 |
| 仓位上限 | 计划最大仓位 |
| 止损条件 | 价格、形态、情绪、板块走弱 |
| 止盈条件 | 一致高潮、放量滞涨、跌破均线等 |
| 风险预案 | 不符合预期怎么办 |
| 复盘状态 | 未复盘、已复盘、已归档 |

### 7.14.3 规则

- 没有交易计划的成交在复盘中自动标记“无计划交易”。
- 计划必须关联至少一个市场阶段判断。
- 计划必须填写止损条件，否则不能标记为“可执行计划”。
- 计划执行后必须进行后验验证。

## 7.15 P15 个人操作复盘日志

### 7.15.1 功能目标

完整记录“我为什么买、为什么卖、是否按计划、错在哪里、下次怎么改”。

### 7.15.2 日志结构

| 模块 | 字段 |
|---|---|
| 基本信息 | 日期、账户、股票、方向、数量、价格、仓位 |
| 市场背景 | 周期阶段、情绪分、指数状态、主线题材 |
| 交易前计划 | 原计划、入场条件、止损、止盈 |
| 实际操作 | 买入、卖出、加仓、减仓、做 T |
| 心态记录 | 贪婪、恐惧、犹豫、冲动、后悔、自信膨胀 |
| 纪律检查 | 是否按计划、是否止损、是否追高、是否重仓 |
| 后验结果 | 1/3/5 日后表现、计划有效性 |
| 复盘结论 | 做对、做错、改进动作 |

### 7.15.3 自动归因

- 周期错判。
- 题材错判。
- 龙头地位错判。
- 入场位置错误。
- 卖点执行错误。
- 情绪化交易。
- 仓位失控。
- 止损拖延。

# 8. 核心用户故事 User Stories

## 8.1 市场复盘类

1. 作为短线交易者，我希望每天收盘后看到市场情绪总分，以便判断明天是进攻、观察还是防守。
2. 作为短线交易者，我希望看到情绪分的分项来源，以便知道分数是由涨停增加拉动，还是由亏钱效应减少拉动。
3. 作为复盘研究者，我希望点击某个交易日看到当日涨停、跌停、连板、主线和龙头，以便复原当天盘面。
4. 作为交易者，我希望系统识别当前周期阶段，以便避免在退潮期重仓接力。
5. 作为交易者，我希望看到情绪高潮风险提示，以便避免在一致加速后追高。

## 8.2 题材主线类

1. 作为龙头战法用户，我希望看到今天最强主线和次主线，以便聚焦资金合力最大的方向。
2. 作为复盘用户，我希望看到题材从试错、发酵、高潮到退潮的变化，以便理解题材生命周期。
3. 作为短线交易者，我希望系统识别一日游题材，以便避免把短期套利误判为主线。
4. 作为研究员，我希望能手工合并或拆分题材，以便修正机器分类错误。

## 8.3 龙头分析类

1. 作为龙头战法用户，我希望看到市场总龙、板块龙和补涨龙，以便判断核心与后排。
2. 作为龙头战法用户，我希望看到龙头竞争关系，以便理解谁在卡位、谁在掉队。
3. 作为交易者，我希望看到龙头分歧和一致状态，以便理解风险收益位置。
4. 作为复盘用户，我希望看到龙头退位原因，以便沉淀失败经验。

## 8.4 趋势战法类

1. 作为趋势交易者，我希望筛选站上长期均线并突破平台的股票，以便寻找趋势龙头候选。
2. 作为趋势交易者，我希望看到个股 RS 强度排名，以便判断它是否强于市场和行业。
3. 作为趋势交易者，我希望同时看周线确认和日线操作信号，以便进行波段管理。
4. 作为趋势交易者，我希望系统提示趋势破坏风险，以便及时复盘持仓。

## 8.5 个股相关性类

1. 作为交易者，我希望输入股票代码后看到它与大盘情绪的相关性，以便判断它是顺势共振还是逆势独立。
2. 作为交易者，我希望系统告诉我个股在所属题材中的地位，以便避免买入后排跟风。
3. 作为交易者，我希望看到个股上涨但情绪和板块走弱的风险背离，以便控制追高风险。

## 8.6 交易计划与复盘日志类

1. 作为交易者，我希望盘前创建交易计划，以便盘中不凭情绪乱操作。
2. 作为交易者，我希望系统自动把成交与计划匹配，以便复盘是否按计划执行。
3. 作为交易者，我希望记录买入和卖出时的心态，以便发现自己是否贪婪、恐惧或冲动。
4. 作为交易者，我希望系统统计我的止损拖延次数，以便改进纪律。
5. 作为交易者，我希望每周看到自己的行为偏差报告，以便减少重复错误。

## 8.7 团队与合规类

1. 作为团队管理员，我希望查看团队成员的复盘完成率，以便推动复盘纪律。
2. 作为研究员，我希望把某段行情保存为案例，以便后续培训和复盘。
3. 作为合规管理员，我希望外发报告前检查敏感表达，以便避免构成投资建议。

# 9. 关键业务流程

## 9.1 每日收盘自动复盘流程

1. 15:00 后触发行情数据同步。
2. 15:10 计算指数、成交额、涨跌分布。
3. 15:15 计算涨停板、连板、炸板、跌停、大面。
4. 15:20 计算题材热度和主线强度。
5. 15:25 计算龙头评分和梯队结构。
6. 15:30 计算情绪总分和周期阶段。
7. 15:35 生成风险标签和系统解释。
8. 15:40 发布当日复盘快照。
9. 用户填写个人复盘。
10. 系统生成个人日报和次日关注点。

## 9.2 盘前计划流程

1. 用户打开昨日复盘。
2. 系统展示当前周期阶段、主线、龙头、风险。
3. 用户选择关注题材和个股。
4. 用户填写入场条件、仓位、止损、止盈。
5. 系统校验是否与周期阶段冲突。
6. 用户确认计划。
7. 盘中成交后自动匹配计划。

## 9.3 盘后个人复盘流程

1. 导入或手工录入当日成交。
2. 系统匹配交易计划。
3. 系统展示交易时点对应的市场情绪、题材状态、个股地位。
4. 用户填写主观心态和复盘结论。
5. 系统生成自动归因。
6. 用户确认或修改归因。
7. 系统更新行为画像和案例库。

## 9.4 人工修正流程

1. 用户发现题材归类、龙头识别或周期阶段不准确。
2. 用户提交修正，填写理由和证据。
3. 系统保存原始版本和人工版本。
4. 团队模式下需要管理员审核。
5. 通过后人工版本可用于报告展示，但不得删除系统原始计算结果。

# 10. 页面级验收标准

## 10.1 总体验收

- 任一页面核心结论必须能追溯到指标、数据源、计算时间和规则版本。
- 所有评分结果必须支持历史回看。
- 所有人工修正必须可审计。
- 所有用户交易数据必须权限隔离。
- 所有导出报告必须包含合规声明。

## 10.2 典型验收用例

| 用例 | 输入 | 期望输出 |
|---|---|---|
| 情绪分计算 | 当日行情完整 | 生成 0-100 分总分、分项分、解释 |
| 数据缺失 | 缺失涨停数据 | 情绪分状态为待确认，提示缺失项 |
| 主线识别 | 某题材连续 3 日涨停居前 | 标记为主线，展示龙头与梯队 |
| 一日游识别 | 首日强、次日无持续 | 标记一日游风险 |
| 龙头退位 | 高标断板且次日大跌 | 龙头状态变更为退位，记录原因 |
| 个股相关性 | 输入股票代码 | 输出相关系数、地位标签、背离诊断 |
| 无计划交易 | 成交无计划 | 日志中自动标记无计划交易 |
| 止损拖延 | 触发止损未执行 | 行为画像增加止损拖延次数 |
| 报告导出 | 生成日报 | 包含情绪、主线、龙头、风险、合规声明 |

## 10.3 性能验收

| 场景 | 要求 |
|---|---|
| 首页加载 | 普通用户 P95 小于 2 秒 |
| 情绪分计算 | 单交易日全市场计算小于 3 分钟 |
| 周期图加载 | 250 日数据 P95 小于 3 秒 |
| 个股相关性 | 单股票 120 日分析小于 5 秒 |
| 报告生成 | 日报小于 10 秒，月报小于 60 秒 |

## 10.4 数据质量验收

- 行情数据缺失率超过 1% 时，不得发布正式快照。
- 涨停原因和题材分类允许人工修正，但必须保留原始值。
- 多数据源冲突时，必须展示冲突明细和最终采用规则。
- 数据重算后，需要记录重算版本和影响交易日。

# 13. 报表与导出

## 13.1 日报模板

日报必须包含：

1. 今日市场结论。
2. 情绪总分和阶段。
3. 指数与资金状态。
4. 主线题材和次主线。
5. 龙头与梯队。
6. 风险标签。
7. 明日关注点。
8. 用户个人操作复盘。
9. 合规声明。

## 13.2 周报模板

周报必须包含：

- 本周情绪分曲线。
- 周期阶段变化。
- 题材轮动路径。
- 龙头生命周期变化。
- 本周最强主线。
- 本周最大亏钱效应。
- 用户交易盈亏结构。
- 用户行为偏差 Top 5。
- 下周风险清单。

## 13.3 月报模板

月报必须包含：

- 月度大势结构。
- 牛熊阶段判断。
- 情绪周期完整回顾。
- 主线题材生命周期。
- 趋势龙头表现。
- 龙头案例与失败案例。
- 个人系统执行情况。
- 下月交易纪律改进目标。

# 14. 风控与合规需求

## 14.1 产品合规边界

系统输出必须遵守：

- 只做研究、复盘、记录、指标解释、风险提示。
- 不输出确定性买卖建议。
- 不承诺收益。
- 不以历史回测暗示未来收益。
- 不诱导用户重仓、满仓、借钱、融资交易。
- 不把系统评分包装为绝对正确。

## 14.2 文案红线

禁止出现：

- 必买、必卖、必涨、稳赚、无脑上。
- 明天一定涨、确定性机会、稳赚套利。
- 满仓干、梭哈、闭眼买。
- 保证收益、翻倍计划。

替代表达：

- 可观察。
- 需验证。
- 风险偏高。
- 情绪处于偏强/偏弱区间。
- 与历史某类阶段相似，但不代表未来重复。

## 14.3 风险提示规则

当出现以下情况，系统必须提示风险：

- 情绪高潮且高标缩量一致。
- 退潮期跌停和大面扩散。
- 指数破位且成交额萎缩。
- 主线龙头断板且后排补跌。
- 用户无计划交易。
- 用户触发止损未执行。
- 用户单笔仓位超过计划上限。

## 14.4 隐私保护

- 用户交易日志默认仅本人可见。
- 团队管理员查看日志需用户授权或组织协议授权。
- 导出报告默认隐藏账户金额、持仓成本、真实资金规模。
- 所有查看和导出操作记录审计日志。

# 15. 非功能需求

## 15.1 性能

- 支持全市场日线、分钟线、涨停事件的高并发查询。
- 首页 P95 响应小于 2 秒。
- 情绪计算任务应支持批量重算。
- 历史回放应支持至少 5 年数据。

## 15.2 可用性

- 核心复盘页面可用性不低于 99.5%。
- 数据源异常时允许展示最近一次可用快照，但必须标记“非最新”。
- 计算任务失败必须有重试和告警。

## 15.3 可解释性

- 所有评分必须展示分项和权重。
- 所有阶段判断必须展示证据。
- 所有风险标签必须展示触发规则。
- 所有人工修正必须展示修正原因。

## 15.4 可扩展性

- 支持新增指标。
- 支持新增评分模型版本。
- 支持新增交易市场和指数。
- 支持新增用户自定义复盘模板。

## 15.5 安全

- 账号密码、Token、交易记录必须加密存储或传输。
- 权限必须按用户、团队、角色隔离。
- 管理后台操作必须有审计日志。
- 不得在日志中输出用户敏感交易明细。

# 17. 术语表

| 术语 | 定义 |
|---|---|

| 炸板 | 个股触及涨停后打开涨停 |
| 回封 | 炸板后重新封住涨停 |
| 大面 | 高位接力失败造成较大亏损 |
| RS 强度 | 个股相对市场或行业的强弱指标 |
| 个股执念 | 忽略周期和主线，反复围绕单只股票主观交易 |

# 20. 最终说明

本 PRD 以“全量产品形态”定义系统，不按阶段裁剪。研发落地时可以根据资源排期拆任务，但产品需求本身必须保持完整闭环：

大势判断 -> 情绪评分 -> 资金供需 -> 主线题材 -> 龙头梯队 -> 个股相关性 -> 交易计划 -> 操作复盘 -> 心态纪律 -> 历史案例 -> 规则回测 -> 数据质量 -> 合规审计。

只有这条链路完整，系统才不是普通行情面板，而是围绕《顿悟股道》方法论构建的情绪周期复盘与交易认知训练系统。
